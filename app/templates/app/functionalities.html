Given below is a brief description of various functionalities of this tool

Generate_AND-OR_Tree(Process):

Generates a AND-OR Tree representing a set of processes


Retrieve_Explanation(Item):

Retrieves all processes with the information for the related condition/prevent/result items


Retrieve_Process_Support(Process):

Retrieves a tree (OrTree) expressing the support for given process(process with all relevant items information)


Retrieve_Items_Support(Items):

Retrieves a tree (OrTree) expressing the processes needed for the item


Get_Items_Generators_Set(Items):

Creates a Cartesian product of the processes which have the specified items as results
For example, assume that the needed items are [itemA, itemB], and they are results of the following processes:
itemA: pr1, pr2
itemB: pr40, pr50
Then the result of this procedure would be:
[pr1, pr40], [pr2, pr40], [pr1, pr50], [pr1, pr50]


Get_Tree(ProcessORTree):

Retrieves the tree from the GLOBAL set of pre-generated trees, which matches the process or the provided tree description and the input - processOrTree is the searched process or And/OrTree


Get_All_Processes_With_Items():

Returns all available processes with populated items information


Get_Items_From_Processes(Processes):

Returns all items used in the described processes with populated processes information
