Welcome to Governance Tool

A brief introduction

We take the perspective that the governance processes to be modeled can be presented as a set of interconnected Processes The goal is not only to construct process models but also to explore how a subset of this can be represented in a database and allow for extracting information about governance processes relevant to certain development projects.

How it works

It generates a set of AND-OR trees of governance steps for a set of entities (e.g. projects) selected based on certain attributes (columns, e.g. project category). It marks items that cannot be achieved because there exists no sequence of steps (either because they are missing or due to explicit contradictions).

Producing a tree implies that it does not recognize if a step is reached via different paths. It also means that it handles multiple conditional items (creating the AND) and alternative steps yielding the same result item (creating the OR).

About Database Models

The tool uses various entries in database to come up with the relevant AND-OR tree for a given project. And, so it's imperitive that we have a robust database model. Know more about the database model here.
